# Python API Parity Centers on Reusable Price Tables

Python API parity means stable workflow-level parity with the C++ API, not one-to-one pybind exposure of every C++ symbol or template. The primary Python construction path is a price table factory that returns a reusable price table for pricing, Greeks, persistence, and interpolated IV; direct interpolated-IV solver construction remains as a backward-compatible convenience. The 4D B-spline price-table path is a first-class acceptance path and must have explicit tests and docs, not just generic backend coverage. Experimental C++ APIs such as `mango::simple` are excluded from the parity baseline, Python binding tests verify reachability/conversion/error plumbing rather than numerical correctness, and the stable Python API does not take a hard numpy/dataframe dependency unless a future bulk numeric API justifies it.
